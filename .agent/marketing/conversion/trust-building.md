# Trust Building — Construção de Confiança

## Visão Geral
Engine de construção de confiança em todas as interações com o cliente. Confiança é o ativo mais valioso de um SaaS — sem ela, não há conversão, retenção ou indicação.

## Pilares de Confiança
1. **Transparência** — Preço claro, sem letras miúdas
2. **Segurança** — Dados protegidos, SSL, criptografia
3. **Prova social** — Clientes reais, resultados reais
4. **Suporte** — Humano, rápido, resolutivo
5. **Consistência** — Mesma qualidade em toda experiência

## Elementos de Confiança na Landing
| Elemento | Onde | Impacto |
|----------|------|---------|
| Selo SSL | Footer | Baixo (é esperado) |
| Depoimentos reais | Features/Pricing | Alto |
| Fotos reais | Hero/Depoimentos | Altíssimo |
| Garantia explícita | Pricing/CTA | Muito alto |
| Suporte visível | Header/Footer | Médio |
| Política de privacidade | Footer | Baixo |

## Construindo Confiança no Produto
- Onboarding guiado (não abandona o usuário)
- Suporte dentro do produto (chat, ajuda)
- Dados exportáveis (o usuário é dono dos dados)
- Transparência de preço (sem cobrança surpresa)

## Recuperação de Confiança
Se algo der errado:
1. Pedir desculpas genuíno
2. Explicar o que aconteceu
3. Resolver imediatamente
4. Compensar (mês grátis, desconto)
5. Comunicar prevenção futura
