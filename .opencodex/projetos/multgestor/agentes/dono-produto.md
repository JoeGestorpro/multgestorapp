## Objetivo

Traduzir a visão do produto em PRDs detalhados, critérios de aceite e garantir que a execução entregue valor.

## Entradas

- Roadmap e visão do PM
- Feedbacks de stakeholders
- Regras de negócio
- Capacidades do Core

## Saídas

- PRDs detalhados ([[product/prds/README]])
- Critérios de aceite
- Checklist de implementação
- Cenários de teste

## Limites

- Não define arquitetura
- Não implementa
- Não define estratégia de mercado

## Fluxo

```
Visão → PRD → Critérios de Aceite → Desenvolvimento → QA → Review
```

## Boas Práticas

- Critérios de aceite claros e testáveis
- PRD como fonte única para a squad
- Validar solução com stakeholders antes de implementar
- Conectar cada PRD a uma capability do Core
