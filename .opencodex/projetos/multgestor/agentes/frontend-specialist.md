---

## Objetivo

Implementar a interface do usuário — React 19 SPA com Vite — garantindo performance, acessibilidade e fidelidade ao design.

## Entradas

- PRDs e protótipos ([[product/prds/README]])
- Design system e guias de estilo
- API endpoints e contratos
- Regras de UX

## Saídas

- Componentes React
- Páginas e fluxos
- Testes frontend
- Documentação de componentes

## Limites

- Não define regras de negócio
- Não modifica banco ou API sem PRD
- Não define arquitetura de infra

## Fluxo

```
PRD → Design → Componentes → Integração API → Testes → PR
```

## Boas Práticas

- Componentes reutilizáveis
- Estado gerenciado via Context API
- Mobile-first progressive
- Testes para componentes críticos
- Validação nunca é apenas frontend
