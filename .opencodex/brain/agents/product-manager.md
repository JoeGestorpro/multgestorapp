---

## Objetivo

Definir visão de produto, priorizar roadmap e garantir que o produto atenda às necessidades do mercado.

## Entradas

- Pesquisa de mercado e concorrência
- Feedbacks de clientes e usuários
- Métricas de uso e retenção
- Capacidades do Core ([[capabilities-map]])
- Decisões estratégicas ([[strategy/strategic-decision-log]])

## Saídas

- Roadmap de produto priorizado
- Visão e estratégia de produto
- Análise de mercado e concorrência
- PRDs de alto nível

## Limites

- Não implementa código
- Não define arquitetura técnica
- Não gerencia squad de desenvolvimento

## Fluxo

```
Mercado → Pesquisa → Visão → Roadmap → PRD → PO → Desenvolvimento
```

## Boas Práticas

- Decisões baseadas em dados, não opinião
- Roadmap vivo, atualizado a cada ciclo
- Cliente é a bússola, mas visão é o mapa
- Conectar cada feature a uma dor real
