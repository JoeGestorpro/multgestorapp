# Checklist Pós-Execução

> Verificações obrigatórias APÓS receber o relatório do Executor.

## Relatório recebido

- [ ] relatório do Executor foi lido e compreendido
- [ ] resumo do que foi feito está claro
- [ ] arquivos alterados foram listados
- [ ] o que foi testado está documentado

## Comparação com objetivo

- [ ] resultado corresponde ao prompt original
- [ ] escopo foi respeitado (nada fora foi alterado)
- [ ] critérios de aceite foram atendidos (total ou parcialmente)
- [ ] se parcial: pendências foram registradas

## Riscos

- [ ] riscos identificados pelo Executor foram revisados
- [ ] nenhum risco novo foi introduzido
- [ ] se risco 4 ou 5: auditoria final está agendada

## Decisões

- [ ] novas decisões foram registradas no projeto
- [ ] se global: registrado em memoria/decisoes-globais.md

## Próximos passos

- [ ] pendências foram listadas
- [ ] próxima missão foi definida (se aplicável)
- [ ] se aplicável: missão foi registrada em queue/next-task.md (com confirmação)

## Compactação

- [ ] conversa foi compactada se necessário
- [ ] compactação foi salva no projeto

## Fechamento

- [ ] estado-atual.md foi atualizado
- [ ] sessão pode ser encerrada (ou continuar com próxima missão)
